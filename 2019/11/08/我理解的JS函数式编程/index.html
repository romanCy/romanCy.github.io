我理解的JS函数式编程

什么是函数式编程？

函数式编程是⼀种编程模型，它将计算机运算看做是数学中函数的计算，主要特点是将计算过程分解成多个可复⽤的函数，并且避免了状态以及变量的概念。

函数式编程不是⽤函数来编程，也不是传统的⾯向过程编程。主旨在于将复的函数符合成简单的函数。运算过程尽量写成⼀系列嵌套的函数调⽤

相对于⾯向对象编程关注的是数据⽽⾔，函数式编程关注的则是动作，其是⼀种过程抽象的思维，就是对当前的动作去进⾏抽象。

相对于⾯向过程编程，函数式编程有更好的代码封装能⼒、代码可读性和代码健壮性。

函数式编程的两个基本特征

函数是第⼀等公⺠

第⼀等公⺠是指函数跟其它的数据类型⼀样处于平等地位，可以赋值给其他变量，可以作为参数传⼊另⼀个函数，也可以作为别的函数的返回值。

函数是纯函数

纯函数是指相同的输⼊总会得到相同的输出，并且不会产⽣副作⽤的函数。从纯函数的概念我们可以知道纯函数具有两个特点：

  • 同输⼊同输出
  • ⽆副作⽤(函数内部的操作不会对外部产⽣影响（如修改全局变量的值、修改 dom 节点等))

函数式编程具有两个最基本的运算：合成（compose）和柯⾥化（Currying）

函数式编程与⾯向对象编程

双⽅优点

函数式编程：⽀持闭包和⾼阶函数，闭包是⼀种可以起函数的作⽤并可以如对象般操作的对象；⽽⾼阶函数是可以以另⼀个函数作为输⼊值来进⾏编程。⽀持惰性计算，这就可以在求值需要表达式的值得时候进⾏计算，⽽不是固定在变量时计算。还有就是可以⽤递归作为控制流程。函数式编程所编程出来的代码相对⽽⾔少很多，⽽且更加简洁明了。

⾯向对象编程：它有三个主要特征，分别是封装性、继承性和多态性。类的说明展现了封装性，类作为对象的模板，含有私有数据和公有数据，封装性能使数据更加安全依赖的就是类的特性，使得⽤户只能看到对象的外在特性，不能看到对象的内在属性，⽤户只能访问公有数据不能直接访问到私有数据。类的派⽣功能展现了继承性，继承性是⼦类共享⽗类的机制，但是由于封装性，继承性也只限于公有数据的继承（还有保护数据的继承），⼦类在继承的同时还可以进⾏派⽣。⽽多态性是指对象根据接收的信息作出的⾏为的多态，不同对象接收同⼀信息会形成多种⾏为。

双⽅缺点

函数式编程：所有的数据都是不可以改变的(因为上边说的纯函数)，严重占据运⾏资源，导致运⾏速度也不够快。⾯向对象编程：不是那么⾃由

总结（在我看来）

不论是⾯向对象编程还是函数式编程，如果你⾛了极端，那都是错误的；⾯向对象编程的极端是⼀切都是对象(纯⾯向对象)；函数式编程的极端是纯函数式编程语⾔。 所以 FP 和 OOP 并不互相排斥，这两种编程范式,⼀个抽象过程、⼀个抽象数据，正好是相辅相成的。 前者擅⻓描述是什么，⽽后者擅⻓描述如何做；FP 抽象出函数，OOP 聚合数据，很完美。 OOP 是⼀整套⼤理论，其中是有层次的，FP 提升系统⾥函数的纯度。 FP 也谈不上⾼⼤上，当你⽤ forEach 和 map 这样的 API 的时候你就在⽤ FP 了

所以当你要抽象数据类型，请⽤ OOP，当你要抽象函数，请⽤ FP。
